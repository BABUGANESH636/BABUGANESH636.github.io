FSM Architecture Overview

Why Finite State Machines?

FSMs provide predictable control over complex gameplay behavior, especially in combat-driven games. This approach improves debugging, testing, and long-term maintainability compared to tightly coupled logic.

The project uses a Finite State Machine (FSM) approach to manage player and enemy behavior. Each state is isolated, self-contained, and responsible only for its own logic.

  • State Machine Core: A central controller handles state transitions and ensures that only the active state processes input and logic.
  • Player States: Idle, Move, Attack, Dash, Hurt, and Death states are implemented as modular units rather than monolithic scripts.
  • Enemy States: Enemies operate using state-driven behavior such as Patrol, Chase, Attack, Stagger, and Death.
  • Scalability: New abilities and behaviors can be added without modifying existing states, reducing coupling and regression risk.
